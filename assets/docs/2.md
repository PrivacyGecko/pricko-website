- Use quarters not exact dates (flexibility)
- Update both sites when status changes
- Be transparent about delays
- Show what's already delivered (proof)

## üìù Content Style Guide

### Language Guidelines

**‚úÖ DO:**
- Use active voice ("Protect your privacy" not "Privacy is protected")
- Keep sentences short and scannable
- Use "you" and "your" (conversational)
- Explain technical terms simply
- Bold key facts for scannability
- Use lists and bullets for clarity

**‚ùå DON'T:**
- Use jargon without explanation
- Promise guaranteed returns or profits
- Make unverifiable security claims
- Use all caps (except $PRICKO in specific contexts)
- Use excessive exclamation marks!!!
- Sound like a robot ("We are pleased to announce...")

### Token-Related Language

**‚úÖ Safe Language:**
- "Utility token"
- "Powers the ecosystem"
- "Unlock features"
- "Governance rights"
- "May fluctuate in value"
- "Not financial advice"

**‚ùå Avoid (Regulatory Risk):**
- ~~"Investment opportunity"~~
- ~~"Guaranteed returns"~~
- ~~"Passive income"~~
- ~~"To the moon"~~
- ~~"Get rich"~~
- ~~"Dividends"~~
- ~~"Profit sharing"~~

### Privacy-First Language

**When discussing privacy:**
- Emphasize user control
- Be specific about what data is/isn't collected
- Explain encryption simply
- Acknowledge trade-offs honestly
- Focus on benefits, not fear

**Example:**
‚ùå "The internet is watching you! Be afraid!"  
‚úÖ "Your browsing data is valuable. We help you keep it private."
